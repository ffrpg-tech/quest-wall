# Contributing

Thanks for wanting to help improve the Farm RPG Quest Wall Calculator! This
is a fan project — not affiliated with FarmRPG, and it will never be
monetized. There are a few ways to contribute, depending on how comfortable
you are with git.

## Contributing code

Bug fixes, UI improvements, and small features are welcome as pull requests.
Bigger changes (new pages, new calculation behavior, anything touching how
quest/item data is fetched or cached) are best discussed in an issue first
so we don't end up with wasted work.

### Local setup

```
npm install
npm run dev
```

The app fetches `questlines.json`/`items.json` from `static/` at runtime
rather than importing them, and those two files are gitignored (see the
data pipeline note below) — so a fresh clone won't have them. `npm run dev`
and `npm run test` both run a `predev`/`pretest` step (`npm run seed-data`)
that copies the committed `static/questlines.sample.json` /
`static/items.sample.json` into `static/questlines.json` / `items.json` if
those files don't already exist. The sample data is fake (made-up quest and
item names) — just enough shape to exercise the app and calculation logic
locally. It never overwrites real data: if you've already run the `api/`
fetch script and have real files in `static/`, the seed step is a no-op.

### Before opening a PR

```
npm run lint     # prettier + eslint
npm run check    # svelte-check
npm run test     # unit tests (vitest) + e2e (playwright)
```

All three should pass. `npm run format` will fix most lint issues
automatically.

### What's out of scope for a PR

- Anything in `static/questlines.json` / `static/items.json` — these are
  generated by the gitignored `api/` fetch pipeline, not hand-edited. Use the
  in-app **Feedback / report an issue** button to report wrong data instead
  of opening a PR that touches those files.
- Monetization, ads, or tracking of any kind — this project won't ship
  those.
